# Finn – Personal Finance Coach

## Overview

Finn is a personal finance web application that I built while learning AI-assisted software development using Replit.

The idea behind the project was to create a simple application that helps users build better spending habits instead of only tracking expenses. Users can create a monthly budget, divide it into different spending categories, monitor their savings, and receive insights about their financial health.

This project gave me practical experience in designing an application, solving development problems, and improving the user experience through continuous testing and iteration.

## Features

- User onboarding and profile setup
- Monthly income and budget planning
- Category-wise budgeting
- Dashboard with spending and savings overview
- Financial health score
- Spending insights
- Monthly comparison page
- Clean dark-themed interface

## Why I Built This

I have always been interested in personal finance and wanted to build a project that solves a real problem.

Many budgeting apps focus mainly on recording expenses. I wanted Finn to encourage users to build better financial habits by showing them how their spending affects their overall financial health.

The project also helped me learn how AI can be used as a development tool while still requiring planning, debugging, and testing from the developer.

---

## How I Built It

I developed Finn using Replit with AI-assisted coding.

Instead of writing every line of code manually, I used AI to help generate code, suggest improvements, and fix errors. I then tested the application, modified the generated code where needed, and refined the features until they worked as expected.

This project helped me understand that AI is most useful when combined with problem-solving and careful testing.

---

## Biggest Challenge

The most difficult part of this project was designing the user interface.

Getting the layout, colours, spacing, and navigation to feel smooth required many small changes. I spent a lot of time adjusting individual components to make the application easier to use and more visually consistent.

Although these changes were small, they made a noticeable difference to the overall experience.
